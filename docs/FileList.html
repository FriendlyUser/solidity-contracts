FileList

contract FileList

Since I can't use an bytes32 array in solidity, I decided to have tag numbers corresponding to string names, almost like a dictionary look up., Stores a filelist on solidity, since an array of structs cannot easily be returned the structs must be decomposed on the front-end.

Source: FileList.sol
Author: David Li

    Main event for smart contract, needed for drizzle to update list of files.

    _filename - bytes32

Functions

  • addFile

    function addFile(string ipfshash, bytes32 _filename, bytes32[] tags) public

    Add a file to the list, updates mappings todos and lastIds.

    Parameters:
    ipfshash - an ipfshash returned after an image is finished uploaded
    _filename - name of file as a bytes32 as filenames should be short
    tags - array of bytes32 used for sorting/searching files (e,g. blockchain, school, textbook)

  • getFileTags

    function getFileTags(address owner, uint256 _index) external view returns (bytes32[])

    Return the tags for a specific file.

    Parameters:
    owner - --- address of person who uploaded the file
    _index - --- the file desired, (first file uploaded, second , etc ...)
    Returns:
    bytes32[]
